## 1. 设计原理：为什么这么搭

### 1.1 顶层哲学：动态流水线，不是静态图书馆

知识不是"拥有"，是"周转"。低周转知识堆积 = 心智被无效信息消耗 = 没想法的根源。

**3 个反常识认知：**

1. **你建了 118 张卡从来不打开**——这不是卡片质量问题，是**使用问题**
2. **LLM 是推理代理，不是索引器**——Concepts 层建完不用 = 白干
3. **删除不是放空**——是主动选择高质量输入，为深度加工留出时间

→ 完整论证：[[知识周转率闭环系统]]

### 1.2 物理布局：WSL + D 盘分工

| 位置 | 角色 | 谁在用 |
|---|---|---|
| **D 盘 `/mnt/d/Obsidian/llm-wiki/`** | 唯一真身，131 个文件，Obsidian vault | 你日常 Obsidian 客户端打开 |
| **WSL `/home/aaron/wiki/`** | 旧 git 仓 + 软链入口 | Hermes skill 工作台，git 备份 |
| **WSL `/home/aaron/wiki/灵感库/`** | **软链** → D 盘 `灵感库/` | `para-collide` 等 skill 写入 |

**为什么这么分：**

- ✅ D 盘有 `.obsidian/` 客户端配置 + 你日常双链笔记
- ✅ WSL 原生 fs 快 10×，git 备份天然有
- ✅ **软链**让两边共享同一份文件，零双写

### 1.3 三环迭代：触发→验证→演进

```
[第一环] 入口触发
  灵感库入盒（4 条入盒规则）
  不满足 → 直接删，不进库
       ↓
[第二环] 验证加工
  Skills 流水线：insight-forge → kuangcai → 输出
  门禁 4 问（两周内必须完成）
       ↓
[第三环] 认知演进
  内容输出 + 上期回顾（认知迭代线）
       ↑________________________↓
           反馈闭环
```

## 4. 维护规则（容易踩的坑）

### 4.1 软链相关

| 操作 | 怎么干 | 风险 |
|---|---|---|
| 在 WSL 写文件 | `touch /home/aaron/wiki/灵感库/...` | 0（自动同步 D 盘）|
| 在 D 盘改文件 | 正常用 Obsidian 编辑 | 0（WSL 端立刻可见）|
| 删软链 | `unlink /home/aaron/wiki/灵感库` | ⚠️ 删了软链 WSL 端访问失败，**不会**影响 D 盘数据 |
| 软链断了 | 重新 `ln -s` | 0 |
| D 盘路径变了 | 改软链 target 即可 | 0 |

### 4.2 PARA 库规则

- ✅ **00-灵感库** 新素材必须满足 4 条入盒规则之一，不满足直接删
- ✅ **门禁 4 问** 两周内必须完成，没过就删
- ✅ **素材不删**（留作以后对撞）
- ❌ **不要在 02-领域/ 堆"未压缩"素材**（那是归档区不是缓冲）
- ❌ **不要新建 05-Skills/**（跟 `~/.hermes/skills/` 重复）
- ❌ **不要新建 02-Areas/**（Synthesis 已在扮演）

### 4.3 跟 git 备份的关系

WSL `/home/aaron/wiki/.git/` 跟踪的是根目录文件。**软链是文件，不进 git**——但软链指向的 D 盘文件如果 D 盘有 git 仓（`/mnt/d/Obsidian/llm-wiki/.git/`）会**双端 git 跟踪**。

**建议：** D 盘 vault 单独 git 仓就够了，WSL 根的 git 仓主要管 `任务控制中心.md` `log.md` 这些纯 WSL 文件。

### 4.4 跟 11 skill 的关系

**Skill 是"给 AI 的执行协议"**，**vault 是"给人看的知识库"**。两者 80% 内容重叠但不冗余：
- Skill：什么时候触发 / 怎么做 / 自检清单
- Vault：为什么这么做 / 哲学依据 / 双链网

**改 skill 必查 `kuangcai/references/pipeline-overview.md` 这张图。**

## 5. 故障排查

### 5.1 para-collide 收件失败

```bash
# 检查软链
ls -la /home/aaron/wiki/灵感库

# 应该看到：灵感库 -> /mnt/d/Obsidian/llm-wiki/灵感库
# 如果没有，重新建：
rm -rf /home/aaron/wiki/灵感库  # 删旧目录（会丢数据，但 D 盘还有）
ln -s /mnt/d/Obsidian/llm-wiki/灵感库 /home/aaron/wiki/灵感库
```

### 5.2 Obsidian 看不到新文件

D 盘 Obsidian 默认会索引新文件。如果 5 分钟还没看到：
1. Obsidian → 设置 → 文件与链接 → 强制刷新
2. 检查文件是不是在 vault 根目录外
3. 检查 `.obsidian/app.json` 里 `alwaysUpdateLinks` 是不是 true

### 5.3 D 盘 WSL 挂载失败

```bash
# 检查挂载
mount | grep /mnt/d
# 应该看到：/dev/sdXX on /mnt/d type ntfs ...
# 如果没有：
sudo mount -t drvfs D: /mnt/d -o metadata,uid=1000,gid=1000
```

### 5.4 skill 写入失败（权限问题）

```bash
# 测试 D 盘可写
touch /mnt/d/Obsidian/llm-wiki/test.md && rm /mnt/d/Obsidian/llm-wiki/test.md
# 如果失败，检查 D 盘目录权限
ls -la /mnt/d/Obsidian/llm-wiki/ | head -5
```

## 8. 给未来 AI agent 的一段话

如果你是一个新接手的 AI agent，读到这份说明书：

1. **所有写文件都通过 WSL `/home/aaron/wiki/灵感库/`**——这是软链，写入会自动落到 D 盘

5. **收到"skill 输出"而非 SKILL.md 的内容时**——识别为下游格式，不装，bridge 进现有 skill
6. **不可逆操作（删文件/推 git）前必须 stop + 3 选项 + 推荐**——参考 `release-changelog-workflow.md`

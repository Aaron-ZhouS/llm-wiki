## 日常 3 件事

### 1. 我想学一个概念 / 看一篇文章

- 触发词：`"大盛，这个概念帮我看一下：XXX"` 或 `"拆一下：XXX"`
- 我会调 `book-extract`（轻）或 `deep-insight`（重）
- 产出落到 D 盘 `灵感库/00-灵感库/` 或 `Synthesis/`

### 2. 我有个模糊想法 / 撞灵感

- 触发词：`"碰一下，我在想 XXX"`
- 我会调 `para-collide`，从 D 盘 `灵感库/00-灵感库/` 拉 2-3 个种子（财富/AI/学习/自媒体/成长）对撞
- 对撞出洞见 → 写卡 → 归档到 `灵感库/02-领域/`

### 3. 我要拍视频号 / 写公众号

| 我想做 | 触发词 | 我跑哪条流水线 |
|---|---|---|
| 30-40 秒视频号短稿 | `"帮我写口播：主题 XXX"` | `book-extract` → `kuangcai` → `video-script-generator` |
| 3-7 分钟 B 站长稿 | `"写个长口播：主题 XXX"` | `deep-insight` → `oral-alchemy` |
| 一张知识卡片 | `"出图：洞见 XXX"` | `knowledge-card-forge` → `knowledge-card-design`（Kindle 纸书风）|
| 一张信息图 | `"出信息图：主题 XXX"` | `sn-infographic` |
| 高密度金句 | `"挖一下：模糊想法 XXX"` | `kuangcai`（挖矿，8 规则压缩）|

## 我不该干的事

- ❌ **不要在 WSL 直接写文件**——绕过软链会破坏 git 跟踪
- ❌ **不要新建 `02-Areas/` 目录**——`Synthesis/` 已经在扮演这个角色
- ❌ **不要新建 `05-Skills/` 目录**——跟 `~/.hermes/skills/` 重复
- ❌ **不要在 00-灵感库 堆 100+ 条素材**——这违背知识周转率，每两周清一轮
- ❌ **不要跳过门禁 4 问直接产出口播**——质量塌方的最快路径

## 出问题怎么查

| 现象 | 一句话诊断 |
|---|---|
| Obsidian 看不到新文件 | 5 分钟还没看到 → Obsidian 设置 → 强制刷新 |
| para-collide 收件失败 | 软链断了：`unlink /home/aaron/wiki/灵感库` 然后 `ln -s /mnt/d/Obsidian/llm-wiki/灵感库 /home/aaron/wiki/灵感库` |
| D 盘挂载掉了 | `sudo mount -t drvfs D: /mnt/d -o metadata,uid=1000,gid=1000` |
| 出图失败 | 改用 `infographic-maker` 或 `baoyu-article-illustrator` 替代 |

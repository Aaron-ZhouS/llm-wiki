## ❶ Cookie 接收链路

### 用途
你 Edge 浏览器用 Cookie-Editor 导出的 JSON → 自动转 yt-dlp 用的 Netscape 格式 → 保存到 `~/.config/yt-dlp/<domain>Cookies.txt` → 后续 yt-dlp / youtube-content / video-link-reader 都能用登录态访问对应网站。

### 怎么用（3 步）

**Step 1**：Edge 装 **Cookie-Editor** 扩展（edge://extensions/ 搜 "Cookie-Editor"）

**Step 2**：在目标网站登录（**别关页面**），点 Cookie-Editor 图标：
- **Export** → 格式选 **JSON**
- ☐ **Include HTTP-only cookies** ← **必勾**
- 复制到剪贴板

**Step 3**：把 JSON 粘到对话里，**说一句"这是 X 站 cookie"**（让 AI 知道域名）

### 触发词

- 「粘 cookie」「cookie 给你」「这是 B站 cookie」
- 「cookie 失效了」「重新登录」
- 用户直接粘 JSON 文本

### 关键路径

| 文件 | 作用 |
|---|---|
| `~/.hermes/scripts/cookie_ingest.py` | 转换脚本（JSON → Netscape）|
| `~/.hermes/skills/mcp/cookie-ingest/SKILL.md` | skill 文档 |
| `~/.config/yt-dlp/bilibili.comCookies.txt` | B站 cookie（当前有效期 2026-06-26）|
| `~/.config/yt-dlp/doubao.comCookies.txt` | 豆包 cookie |

### 验证命令

```bash
# 查所有 cookie 文件状态 + 过期时间
python3 ~/.hermes/scripts/cookie_ingest.py --verify
```

### 现状

- ✅ **B站 cookie**：24 条，**剩 3 天**（6-26 过期）
- ✅ **豆包 cookie**：31 条，**剩 27 天**（7-20 过期）
- ❌ **YouTube cookie**：未配置（等需要时再导）
- ❌ **抖音 cookie**：已过期 42 天（WSL 反正不通，弃用）

### ⚠️ 安全注意

- Cookie = 账号登录凭证，**只发给你的 AI**，绝不外传
- `chmod 600` 自动设置（脚本内置）
- cookie 失效是正常的（几天到 1 个月），失效后用 `--verify` 查再重导

## ❷ 决策百科

### 用途
记录**做过的关键决策** + **当时怎么想的** + **半年后回头验证**。下次再面对类似决策，调出自己过去的判断来参考（**不踩自己踩过的坑**）。

### 跟"对话日志"的区别

| 维度 | 对话日志 | 决策百科（这版）|
|---|---|---|
| 记录对象 | 跟 AI 聊过什么 | **我做过什么选择** |
| 价值 | 低（不查聊天）| **高（决策辅助）** |
| 触发词 | "记得我们说过..." | "我上次是怎么决定的..." |

### 什么时候记一条（任一即触发）

- [ ] 方向选择（赛道 / 主业 / 副业 / 退出某个事）
- [ ] 合作选择（跟谁合作 / 拒绝谁的邀请）
- [ ] 资源投入（花钱 / 投入时间超过 1 周 / 装修某个工具链）
- [ ] 对外承诺（答应别人的事 / 公开说出去的观点）
- [ ] **拒绝选项**（主动放弃什么 — 最容易忘，反而最该记）

### 6 字段模板

```
① 决策时点
② 当时考虑的选项（≥ 2 个）
③ 最终选了哪个 + 为什么（按重要性排序）
④ 预判风险（当时觉得会出什么问题 + 应对）
⑤ 半年后复盘 ← 半年后回来填这节
⑥ 下次类似决策怎么参考
```

## ❹ GitHub 配置备份

### 用途
**两层备份**：
- **A 本地层**（D 盘 bare 仓库）— 立即生效，防 WSL 重装
- **B 远程层**（GitHub 私有仓库）— 防电脑暴毙，等你贴 token 激活

### 备份内容

✅ 备份：
- `SOUL.md`（AI 身份/语气）
- `config.yaml`（主配置）
- `memories/`（MEMORY.md + USER.md）
- `skills/`（所有 skill 源码）
- `profiles/`（4 个 profile）
- `cron/jobs.json`（定时任务）
- `scripts/`（自定义脚本）

❌ 绝不备份：
- `.env`（含 API key）
- `cache/` `backups/` `audio_cache/`（临时数据）
- `*.bak` `*.tmp`（历史/锁文件）
- 认证 / 会话状态（auth.json / channel_directory.json 等）

### 怎么用

```bash
# 第一次：初始化本地仓库
~/.hermes/scripts/hermes-backup.sh --init

# 日常：跑备份（rsync + git commit + 推本地 bare）
~/.hermes/scripts/hermes-backup.sh

# 推 GitHub 远程（需先配 remote）
~/.hermes/scripts/hermes-backup.sh --remote

# 看状态
~/.hermes/scripts/hermes-backup.sh --status
```

### 触发词

- 「备份 Hermes」「把配置备份了」
- 「我的配置在哪」

### 当前 commit 历史

```
39d497a  ops-discipline skill + MEMORY + index.md Skills
48eab47  hermes core backup (SOUL + config + cron + memories + scripts)
34f27e5  init
```

### ⚠️ 远程层待你激活（3 步）

1. Edge 开 github.com → New repo → 私有仓库 `hermes-config-backup`
2. Settings → Developer settings → Personal access tokens → **Fine-grained**
   - Repository access: 仅 hermes-config-backup
   - Permissions: **Contents R/W**（其他都给 No）
   - 过期：建议 30 天（视频说永不过期，**走 30 天更安全**）
3. 复制 token 粘给我 → 我加 remote + 设每日 23:00 push

### 关键路径

| 文件 | 作用 |
|---|---|
| `~/.hermes/scripts/hermes-backup.sh` | 备份脚本 |
| `/mnt/d/Hermes-Backups/hermes-config.git` | 本地 bare 仓库 |
| `/mnt/d/Hermes-Backups/work/` | work tree（实际文件）|
| `/mnt/d/Hermes-Backups/backup.log` | 运行日志 |
| `~/.hermes/.gitignore` | 排除规则（脚本自动生成） |

### 已知陷阱

- NTFS 文件系统慢，**100M+ git add 经常超时** → 拆目录 add + 后台跑
- 一定要 `--init` 跑一次（不跑的话 `--status` 显示未初始化）

选题的本质不是"找用户感兴趣的话题"，
是去"用户问题的密集区"开店。

⚙

系统运转机制

✗ 默认假设谬误

多数人以为"好内容 = 好选题"，所以抄爆款、让 AI 拆解、复制粘贴。 结果：选题枯竭，账号没有差异化。

⚡ 关键杠杆点

大盛的杠杆：翻评论区 + 私域付费问答。 用户在主动告诉你他们的问题—— 你拍他们的问题，他们就会搜过来。

干完就着 → 抄爆款 → 没资产 → 数据一般
挖评论区 → 一次投入 → 多次搜索 → 资产滚雪球
⚖

资产型 vs 消耗型

资产型 ASSET

一次投入 · 持续产出

  • ✓ 评论区挖出来的高频问题
  • ✓ 用户原话做标题
  • ✓ 四段式结构化（问题→方案→原因→总结）
  • ✓ 沉淀成搜索资产（一直有流量）

→ 加大投入 · 时间复利

消耗型 COST

干完就着 · 下次从零

  • ✗ 抄爆款、让 AI 拆解爆款公式
  • ✗ 追平台热词、追风口话题
  • ✗ 高大上的钩子、抽象的话题
  • ✗ "赚够 100 万怎么花"这类离用户远的话题

→ 砍掉 · 找用户问题

🚧

边界条件

成立前提 · 评论区有活跃度（粉丝 ≥ 100、有真实评论可挖）；粉丝画像相对集中。
断裂点 · 冷启动期账号（评论 < 50 条） / 一次性 campaign 大促 / B 站深度长视频。
◆

最小可执行动作 · 评论区四问

（每周日晚上）
  1. ①
    翻 100 条评论 — 看你最近的所有视频/文章的评论区
  2. ②
    按问题归类 — 这个用户问的是什么，那个问的又是什么
  3. ③
    数频次 — 前 5 个问题 = 你下周 5 条视频的选题
  4. ④
    四段式拍 — 问题 → 方案 → 原因 → 总结

每一条视频都是

"可以复用"

的资产。

# 选 Hermes 作为唯一 AI 主力

## ① 决策时点

2026-05-21

## ② 当时考虑的选项

- **选项 A**：Hermes Agent（主力）+ 零碎小工具
- **选项 B**：QClaw（多模态国产 Agent）
- **选项 C**：Codex / Claude Code（代码 IDE 风格）
- **选项 D**：Coze / 字节扣子（低代码 Agent 平台）

## ③ 最终选了哪个 + 为什么

**选了 A Hermes 唯一主力**

理由：
1. **偏好简单系统**（个人原则 — 不装多余工具）
2. Hermes 已经有飞书 + 微信双通道（移动端 OK）
3. **任务可拆解**：用户→Hermes（拆解/调度/验收）→ DeepSeek V4 Flash + Aider（执行代码）
4. Codex CLI 在 WSL 跑不通（已验证 hang），不浪费时间

## ④ 预判风险

- **Hermes 升级/维护**依赖单一项目 → 应对：核心 skill 自己掌握（mcp/cookie-ingest / knowledge-card-design 等固化在本地）
- **DeepSeek V4 Flash 余额 9.99 元**（已记录）→ 应对：模型 fallback 列表记好
- **跨平台同步**（Windows ↔ WSL）有时卡 → 应对：核心配置放 ~/hermes/

- **AI 工具选型核心 = 简单 > 强大**。一人公司没精力维护多套工具链

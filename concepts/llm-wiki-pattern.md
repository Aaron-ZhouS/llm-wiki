# LLM Wiki Pattern

## 定义

LLM Wiki Pattern（LLM维基模式）是一种用大语言模型构建和管理个人知识库的方法论，核心思想是让AI增量构建和维护一个持久的、结构化的、可交叉引用的Markdown文件知识库。

与 RAG（检索增强生成）不同，Wiki 不是在每次查询时从原始文档重新发现知识，而是**知识只编译一次，之后直接使用**。

## 核心特征

- **持久积累**：知识编译一次后持续保持最新，无需每次查询重新推导
- **交叉引用已建立**：相关概念之间已建立链接，查询时直接定位
- **矛盾已标注**：新旧信息冲突时，AI 会标注而非静默覆盖
- **复合效应**：每添加一个新来源，知识库自动更新相关页面，产生复合增长

## 三层架构

```
Raw Sources（原始来源层）
    ↓ LLM 读取但从不修改
Wiki（维基层）← LLM 全权维护
    ↓ LLM 全权维护
Schema（规范层）← 定义结构和约定
```

- **Raw Sources**：原始文档（文章/PDF/播客笔记），不可变
- **Wiki**：LLM 生成的实体页、概念页、对比页、摘要页
- **Schema**：规范文档（如 SCHEMA.md），定义页面结构、标签体系、约定

## 操作类型

| 操作 | 说明 |
|------|------|
| **Ingest** | 添加新来源到 raw/，LLM 分析并更新相关 wiki 页面 |
| **Query** | 对 wiki 提问，LLM 搜索相关页面并综合回答 |
| **Lint** | 定期健康检查：矛盾/孤立页面/过期内容/断链 |

## 应用场景

- **个人成长**：追踪目标、健康、心理，积累文章和笔记
- **AI研究**：深入研究某个主题，阅读论文并增量构建知识网络
- **一人公司**：内容创作流程、竞品分析、客户沟通记录
- **阅读书籍**：每读一章建立笔记页面，构建书籍伴读维基

## 关键分工

> "Obsidian 是 IDE；LLM 是程序员；wiki 是代码库。"

- 人类负责：来源 curation、方向把控、提问
- LLM 负责：总结、交叉引用、归档、维护一致性

## 与 RAG 的区别

| | RAG | LLM Wiki |
|---|---|---|
| 知识状态 | 每次查询重新发现 | 一次编译，持续保持 |
| 积累性 | 无，问过就忘 | 有，越用越丰富 |
| 矛盾处理 | 静默覆盖 | 显式标注 |
| 适用规模 | 小文档集 | 中大规模知识体系 |

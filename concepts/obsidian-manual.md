## 1. Obsidian 是什么

Obsidian 是一款**本地优先的笔记软件**，基于 Markdown 文件，所有笔记存在本地文件夹里，不会被锁定在任何平台。

**本 Wiki 的特点：**
- 你的笔记通过 GitHub 在 Windows/iPad/AI（我）三端同步
- 所有笔记都是纯文本 Markdown 文件，永不丢失
- 支持双向链接，形成知识网络

**核心概念：**
- **Vault（保险库）**：一个文件夹，包含所有笔记
- **Markdown（.md）**：纯文本格式，支持 `[[双向链接]]` 和 `#标签#`
- **双向链接**：类似"超链接"，但会自动追踪引用关系

## 2. 首次打开设置

### Step 1：打开 Vault

1. 打开 Obsidian
2. 点击 **Open another vault**
3. 选择 **Open folder as vault**
4. 找到路径 `D:\Obsidian\llm-wiki`
5. 点击 **打开**

> 首次打开可能会提示"安全模式"，点击 **Turn off Safe Mode**（关闭安全模式），然后去装插件。

### Step 2：安装必要插件

Obsidian 需要安装 **Obsidian Git** 插件才能实现自动同步：

1. 点击左侧 **设置**（齿轮图标）⚙️
2. 左侧菜单 → **社区插件**
3. 如果提示"安全模式"，点击 **Turn off safe mode**
4. 点击 **浏览** 按钮
5. 搜索 `Obsidian Git`
6. 点击 **Install** → 等待安装完成
7. 点击 **启用**（Enable）

### Step 3：配置 Obsidian Git 自动同步

1. 左侧菜单 → 找到 **Obsidian Git**（在"社区插件"下方）
2. 打开以下选项：
   - ✅ **Auto commit** — 自动提交更改
   - ✅ **Auto pull** — 自动拉取远程更改
   - ✅ **Auto push** — 自动推送到 GitHub
3. **Auto push interval (minutes)** 设置为 `5`

> 配置完成后，Obsidian 会每 5 分钟自动把你的更改推送到 GitHub，AI（我）这边也会同步收到。

### Step 4：确认同步状态

Obsidian 窗口右下角可以看到 Git 状态图标：
- 🔄 绿色图标 = 正在同步
- ✅ 蓝色图标 = 同步完成
- ⏳ 黄色图标 = 有待推送的更改

## 4. 创建和编辑笔记

### 4.1 创建新笔记

**方法一：快捷键**
1. 按 `Ctrl + N` — 新建笔记
2. 输入文件名，按 `Enter`
3. 开始编辑

**方法二：右键菜单**
1. 在左侧文件列表空白处**右键**
2. 选择 **新建笔记**
3. 输入文件名

**方法三：命令面板**
1. 按 `Ctrl + P`
2. 输入"新建"
3. 选择 **Markdown 格式新建笔记**

### 4.2 基础 Markdown 格式

```markdown
# 这是一级标题
## 这是二级标题
### 这是三级标题

**这是加粗文字**
*这是斜体文字*
~~这是删除线~~

- 这是无序列表
- 条目二

1. 这是有序列表
2. 条目二

> 这是引用块

`这是行内代码`

```
这是代码块
多行代码
```
```

### 4.3 实时预览

Obsidian 支持**一边编辑一边预览**：

- **默认**：编辑模式和预览模式合并（所见即所得）
- 按 **Ctrl + E**：切换到纯预览模式
- 按 **Ctrl + P** → 输入"切换预览"：切换编辑/预览

### 4.4 保存

Obsidian **自动保存**，无需手动保存。修改后几秒内文件自动更新。

## 5. 使用双向链接

双向链接是 Obsidian 最强大的功能，相当于"知识蜘蛛网"。

### 5.1 创建双向链接

**链接到已有笔记：**
在笔记中输入 `[[` 会弹出笔记列表，选择要链接的笔记。

```markdown
[[video-account-rules]]  — 链接到"视频号规则"笔记
[[llm-wiki-pattern]]     — 链接到"LLM Wiki模式"笔记
```

**创建新笔记的同时链接：**
```markdown
[[新笔记名称]]  — 如果新笔记不存在，链接会带"?"号
点击后自动创建该笔记
```

### 5.2 显示双向链接

- 按住 `Ctrl` 键 + 鼠标悬停在链接上 → 预览目标笔记内容
- 点击链接 → 直接跳转到目标笔记
- 右侧 **反向链接面板**（Backlinks）— 显示所有引用当前笔记的文件

### 5.3 链接到文件夹

```markdown
[[raw/papers/]]      — 链接到某个文件夹
[[raw/papers/文件名]] — 链接到具体文件
```

### 5.4 嵌入笔记内容

使用 `!` 前缀可以**嵌入**目标笔记的内容：

```markdown
![[video-account-rules]]  — 在当前位置显示该笔记的完整内容
```

### 5.5 标签（Tags）

```markdown
#标签名称
#wechat #视频号 #运营

嵌套标签：
#区域/北京 #区域/上海
```

> 标签可以在左侧边栏点击查看所有使用同一标签的笔记。

## 6. 文件夹结构

你的 Wiki 目录结构如下：

```
llm-wiki/
├── SCHEMA.md              ← 知识库规范，定义笔记格式标准
├── index.md               ← 知识库索引，查找笔记的入口
├── log.md                 ← 操作日志，记录知识库的每次更新
│
├── entities/              ← 实体页面（人物、工具、品牌）
│   ├── andrej-karpathy.md
│   ├── hermes-agent.md
│   ├── rag.md
│   └── video-account-rules.md
│
├── concepts/              ← 概念页面（方法论、理论）
│   ├── llm-wiki-pattern.md
│   └── video-algorithm.md
│
├── comparisons/           ← 对比分析页面
│   └── rag-vs-llm-wiki.md
│
├── queries/                ← 问题与答案页面
│
└── raw/                    ← 原始资料（未处理的来源）
    ├── articles/           ← 公众号/博客文章
    ├── papers/             ← 论文/研究报告
    └── transcripts/        ← 视频/音频转录稿
```

### 文件命名规范

- ✅ `video-account-rules.md`
- ✅ `llm-wiki-pattern.md`
- ❌ `视频号规则.md`（避免中文文件名，某些系统有问题）
- ❌ `新 笔记.md`（避免空格）

## 7. 搜索和查找

### 7.1 快速搜索

按 **Ctrl + O** 或点击左侧**放大镜图标**打开搜索：

- 输入关键词即时搜索
- 支持搜索文件名和正文内容
- 按 `Enter` 在当前笔记中搜索
- 按 `Ctrl + Enter` 全库搜索

### 7.2 命令面板

按 **Ctrl + P** 打开命令面板，输入命令名称执行操作：

| 命令 | 作用 |
|------|------|
| `新建笔记` | 创建新笔记 |
| `搜索` | 全库搜索 |
| `切换预览` | 编辑/预览切换 |
| `打开设置` | 打开设置面板 |
| `Obsidian Git: Push` | 手动推送同步 |

### 7.3 标签搜索

左侧边栏点击 **标签图标**（#），展开所有标签列表，点击标签查看相关笔记。

### 7.4 Graph 视图

左侧边栏点击 **Graph 视图图标**（一个网络图），可以看到所有笔记的链接关系：
- 每个圆点 = 一篇笔记
- 每条线 = 双向链接关系
- 颜色区分不同文件夹/类型

> Graph 视图是发现知识盲区的好工具——哪些概念没有建立链接，哪些笔记是"孤岛"。

## 8. AI 自动同步原理

理解同步机制，不用担心数据丢失。

### 8.1 同步架构

```
Windows Obsidian
     ↓ (每5分钟自动)
GitHub 仓库
     ↑ (每5分钟自动)
AI (Hermes Agent / WSL)
     ↓ (AI生成/更新笔记)
     ↓
  你在 Windows Obsidian 看到更新
```

### 8.2 手动同步

如果急需同步，可以手动推送：

1. 按 `Ctrl + P`
2. 输入 `Obsidian Git: Push`
3. 按 `Enter`

### 8.3 查看同步状态

右下角状态栏：
- **蓝色圆圈 + 勾** ✅ = 一切同步完成
- **黄色圆圈** ⏳ = 有待推送的更改
- **红色** = 同步出错

### 8.4 冲突处理

如果 Windows 和 AI 同时修改了同一笔记，Git 会保留两个版本：
- GitHub 上会有**冲突标记**
- 需要手动决定保留哪个版本
- 遇到冲突时联系 AI 协助解决

### 8.5 数据备份

你的所有笔记在三个地方有副本：
1. **Windows 本地** `D:\Obsidian\llm-wiki\`
2. **GitHub 仓库** github.com/Aaron-ZhouS/llm-wiki
3. **AI 端（WSL）** `/mnt/d/Obsidian/llm-wiki/`

```markdown

tags: [视频号, 运营技巧, 2026]

---

# 我的笔记正文
```

这会生成一个表格，列出所有带"视频号"标签的笔记。

## 10. 常见问题

### Q1: 笔记有修改但右下角一直显示"有待推送"？

**原因**：Auto push interval 可能设置太长，或者 Git 凭证过期。

**解决方法**：
1. 按 `Ctrl + P`
2. 输入 `Obsidian Git: Push`
3. 按 `Enter` 手动推送一次

如果仍然失败，联系 AI 协助排查。

---

### Q2: iPad 上能看到笔记吗？

**可以。** 在 iPad 上：
1. 安装 Obsidian App
2. 安装 **Obsidian Git** 插件
3. Clone 仓库：`https://github.com/Aaron-ZhouS/llm-wiki`
4. 每次打开 App，按 `Ctrl + P` → `Obsidian Git: Pull` 拉取最新内容

> iPad 版本也可以设置 Auto pull（设置 → Obsidian Git → Auto pull → 打开）

---

### Q3: 想不起某个笔记的文件名怎么办？

1. 按 `Ctrl + O` 打开快速切换器
2. 输入笔记的任意关键词
3. Obsidian 会模糊搜索文件名和正文内容

---

### Q4: 误删了笔记怎么恢复？

**方法**：从 GitHub 历史恢复

1. 打开 https://github.com/Aaron-ZhouS/llm-wiki
2. 找到要恢复的文件
3. 点击该文件 → 点击 **History（历史）**
4. 找到误删前的版本 → 点击 **Browse（浏览）**
5. 点击 **Raw** → 复制内容
6. 在 Obsidian 新建同名笔记，粘贴内容

或者联系 AI 协助从 Git 历史恢复。

---

### Q5: Windows 端改了笔记，AI 多久能收到？

**约 5-10 分钟。**
- Obsidian Git 每 5 分钟自动推送一次
- AI 端 WSL cron 每 5 分钟拉取一次
- 如果急需，联系 AI 执行即时同步

---

### Q6: Obsidian 打开 vault 报错怎么办？

**常见错误及解决：**

| 错误提示 | 解决方法 |
|---------|---------|
| "Vault not found" | 检查路径是否正确，路径中不能有特殊字符 |
| "Cannot open vault" | 用管理员权限打开 Obsidian |
| "Some files not loaded" | 某些文件名含非法字符，重命名文件 |

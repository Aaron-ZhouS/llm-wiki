# 00 - 灵感库 · 入口规则

> 你的系统本质上是**「动态的流水线」**而非**「静态的图书馆」**。
> 知识不是"拥有"，而是"周转"。

---

## 这个目录是什么

**灵感库** = 灵感触发后，**还没想透、还没验证**的笔记。
不是"收藏夹"，不是"待办"，不是"有用的东西先存着"。
是：**让你不舒服的那个问题。**

---

## 入盒规则

进这个库的笔记，必须满足**至少一条**：

- 🔴 这个观点我不认同，我要说出来
- 🔴 这个东西我以为懂了，但讲不清楚
- 🔴 这个现象让我困惑，没有简单解释
- 🔴 这个旧认知最近被动摇了

---

## 出盒规则（强制，两周内必须完成）

> **门禁四问**
>
> 1. 我能在 10 秒内说出这个观点的反例吗？ ← 可证伪测试
> 2. 它能同时解释 A 和 B 两个不相干现象吗？ ← 穿透力测试
> 3. 讲给一个比我聪明的人听，对方会质疑哪里？ ← 最强反方测试
> 4. 我之前有没有讲过类似的？这次新在哪？ ← 迭代检查

**通过 → 提炼进 Skills 加工管线 → 输出**
**没通过 → 删掉**

---

## 管理规则

- 超过两周没处理的笔记 = **直接删，不可惜**
- 定期（每周日）浏览一次灵感库，清一轮
- 库里干净 = 脑子干净

---

## 触发 → 验证 → 演进

不是"积累 → 存储 → 检索"，而是**三环迭代**：

```
[入口触发]  灵感库入盒（4 条入盒规则）
    ↓
[验证加工]  Skills 流水线（insight-forge → kuangcai → 输出）
    ↓
[认知演进]  内容输出 + 上期回顾（认知迭代线）
    ↑________________________↓
           反馈闭环
```

## 核心规则

### 任务状态

| 状态 | 含义 |
|------|------|
| `待分派` | 协调者已定义，等待分配 |
| `协调中` | 协调者正在处理 |
| `调研中` | 研究专家执行 |
| `写作中` | 作家执行 |
| `构建中` | 构建者执行 |
| `待验收` | 等待协调者验收 |
| `完成` | 验收通过，已归档 |
| `阻塞` | 依赖未满足，等待解锁 |

### 验收标准规则

**每个任务必须有量化验收条件**，格式：

```
验收标准：
- [ ] 条件1（可检验）
- [ ] 条件2（可检验）
```

模糊的验收标准 = AI 糊弄的空间。

## Git 安全网

- 每个角色完成一个任务节点后，执行一次 commit
- Commit 格式：`[角色] 任务ID: 完成内容`
- 如果出错，执行 `git reset --soft HEAD~1` 回退到上一个节点

---

## Cron 续命机制

每30分钟检查一次任务文件：
- 有 `进行中` 任务且超过2小时无更新 → 提醒
- 有 `阻塞` 任务 → 报告依赖问题
- 有新任务 → 通知协调者
